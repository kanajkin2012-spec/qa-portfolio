# 🐛 Bug: "First" pagination button does not respond to click

## 📝 Description
When clicking the "First" button in the table pagination, there is no reaction: no page change, no visual feedback.

## 🖥 Environment
- **OS**: Windows 11
- **Browser**: Google Chrome v. 146.0.7680.165
- **URL**: https://demoqa.com/webtables

## 👣 Steps to Reproduce
1. Open https://demoqa.com/webtables
2. Wait for the page to fully load
3. Scroll down to the "Web Tables" section
4. Find the pagination controls below the table
5. Click the "«" button (First)

## ✅ Expected Result
- Table navigates to the first page
- "First" button becomes disabled or highlighted as active
- First records of the table are displayed

## ❌ Actual Result
- Button does not respond to click
- No page change occurs
- No visual feedback

## ⚙️ Severity & Priority
- **Severity**: Major
- **Priority**: High
- **Reproducibility**: 100% (always)
